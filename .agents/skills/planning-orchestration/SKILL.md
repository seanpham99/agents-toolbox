# Planning Orchestration

Turn approved intent into an executable, ordered plan.

## When To Use

- Implementation is approved and needs execution sequencing
- Task has dependencies, milestones, or multiple tools
- Team needs checkpoints and rollback-aware planning

## Procedure

1. Restate target outcome and constraints.
2. Break work into ordered steps with clear ownership.
3. Define prerequisites and blockers for each step.
4. Select tools only required for the current step.
5. Define validation checkpoints after major steps.
6. Identify rollback or fallback path for risky changes.
7. Execute step-by-step and update status as work progresses.

## Output Format

1. Goal
2. Ordered steps
3. Dependencies
4. Validation checks
5. Risks and fallback

## Anti-Patterns

- Do not front-load unnecessary tooling.
- Do not skip validation between major steps.
- Do not mix planning with unresolved requirements.
